Detect knowledge-base gaps by mining daily logs + lessons for repeated questions and learnings that aren't yet captured in the KB.

**Input:** $ARGUMENTS — optional flags:
- `--days N` (default `14`) — scan logs from the last N days
- `--since YYYY-MM-DD` / `--until YYYY-MM-DD` — explicit date range (overrides `--days`)
- `--min-recurrence N` (default `2`) — minimum hit count per topic to flag as gap
- `--repo NAME` — restrict scan to logs containing this repo name
- `--auto` — write proposed KB drafts directly without prompting
- `--verbose` — also dump full report inline in chat

Follows `.claude/docs/workspace-write-guard.md` for applied KB drafts and daily-log apply-count updates. Auto-write modes only skip human confirmation; they never skip final drift checks. `--auto` skips only the prompt, not target staging or `workspace-write-guard.py apply`.

## Output Discipline

The full gap report is written to `workspace/tmp/kb-gaps-{YYYY-MM-DD}.md`. Default chat receives only:
- 1-line summary (`{N} gaps detected → {path}`)
- Top-3 highlights (one line each)

With `--verbose`: also dump the full report inline below the summary.

## Steps

### Preflight — Language (MUST run before Step 1, non-negotiable)

Follow `.claude/docs/language-config.md` → Minimum Step 0 block. The report file `workspace/tmp/kb-gaps-{date}.md` follows `conversation:`; quoted KB content keeps its source language.

### Step 1: Resolve date range

Parse `$ARGUMENTS`. If `--since` / `--until` not given, derive from `--days N` (default 14):

```bash
RANGE=$(python3 .claude/scripts/date-resolve.py "${N}d")
START=${RANGE%% *}
END=${RANGE##* }
```

### Step 2: Run pre-filter scan (deterministic)

Invoke the regex pre-filter — zero-token deterministic stage:

```bash
bash .claude/scripts/kb-gap-scan.sh \
  --since "$START" --until "$END" [--repo "$REPO"]
```

Output is TSV: `marker_type<TAB>file<TAB>line<TAB>snippet`. Capture stdout.

If exit 2 (no hits): emit `No knowledge gaps detected in {range}` and stop. No report file.

### Step 3: Cluster hits by topic

Group raw hits into topic clusters. For each cluster, infer:
- **Topic key** — short noun phrase (≤8 words; e.g., "EF Core OrderBy + Skip ordering")
- **Recurrence** — count of *distinct days* (one log file = one day) the topic appears in
- **Marker mix** — set of marker types triggering it (e.g., `{sme_teach, post_error}`)
- **Evidence** — up to 5 entries: `{file}:{line} — {snippet}`

Drop clusters where `recurrence < min-recurrence` (default 2). One-shot signals are ignored unless the user passes `--min-recurrence 1`.

Heuristics:
- Same repo name + same domain term → likely one cluster
- `lookup`-only with no SME involvement → low confidence; keep but flag

### Step 4: KB cross-check

For each surviving cluster, run:

```bash
bash .claude/scripts/kb-search.sh --with-score "{topic key}"
```

- Exit 2 (no hits) **OR** top score `< 2` → confirmed gap
- Top score `≥ 2` → already covered; drop, or mark "thin coverage" (lower priority — include in report only if the matching entry is `> 90 days` old)

Read the top score from the first `**Score:** N` line. If the score line is missing, treat the result as `thin coverage` rather than confirmed coverage.

### Step 5: Route each gap to a KB target

For each confirmed gap:
- Mentions a specific repo → `bash .claude/scripts/kb-domain-resolve.sh {repo}` → `workspace/kb/projects/{repo}.md`
- Cross-repo / general pattern → pick the closest existing `general/*.md` from `.domain-map.md`; fallback `workspace/kb/general/_inbox.md` (create if missing)
- Ops / runbook topic → closest existing `ops/*.md`; fallback `workspace/kb/ops/_inbox.md`

Confidence:
- `high` — marker mix includes `sme_teach` OR `post_error`
- `medium` — `uncertainty + lookup` mix
- `low` — `lookup`-only or `first_time`-only

### Step 6: Draft entries

For each gap, produce a draft using the standard `/nase:kb-update` entry format:

```
### {today} — {topic}
**What:** {one-line summary inferred from snippets}
**Why it matters:** {hypothesis from marker mix}
**Details:**
- {snippet 1, condensed}
- {snippet 2, condensed}
**Source:** detected by /nase:kb-gap-detect on {today}
**Tags:** gap-detected, {confidence}
**Confidence:** {high|medium|low}
```

Track each draft alongside its proposed target file path.

### Step 7: Write report

Write to `workspace/tmp/kb-gaps-{today}.md`:

```
# KB Gap Report — {today}
Generated by `/nase:kb-gap-detect` over {START}..{END}
({D} log days, {M} hits, {K} clusters, {G} gaps, {T} thin-coverage)

## Gap 1: {topic}  [recurrence: {N}, confidence: {high|medium|low}]
**Markers:** {marker mix}
**Evidence:**
- {file}:{line} — "{snippet}"
- ...
**KB target:** `{path}`
**Existing coverage:** {none | thin: top hit "{title}" score {S}, last entry {date}}

**Draft entry:**
{draft block from Step 6}

---

## Gap 2: ...
```

After the file is saved, hold off on chat output — the bounded summary in Step 9 is the single source of chat content.

### Step 8: Daily log + approval

Append to today's daily log following `.claude/docs/daily-log-format.md` (tag: `kb-gap-detect`):
```
({START}..{END}) — {G} gaps detected, {none-yet|N applied}
```
Write the line **before** prompting so it lands regardless of the user's next choice.

If `--auto` was passed: skip the prompt; apply all drafts; update the log line's apply count.

Otherwise invoke `AskUserQuestion`:
```
question: "{G} gaps detected. Apply drafts to KB?"
header: "KB Gap Detection"
options:
  - label: "Apply all"      , description: "Append every draft to its proposed target file"
  - label: "Pick subset"    , description: "Show numbered list, accept comma-separated picks"
  - label: "Just the report", description: "Skip writes — report stays in workspace/tmp/"
```

For each applied gap:
1. Read the target KB file
2. Stage the updated target under `workspace/tmp/`, show the diff, re-check mtime/hash, then append the draft under the appropriate section header (or create the section if missing)
3. Increment the daily-log apply tally

### Step 9: Chat summary

Print to chat:

```
## KB Gap Detection — {today}
Range: {START}..{END} ({D} log days)
Hits: {M} | Clusters: {K} | Gaps: {G} | Thin: {T}
Top 3:
  1. {topic}  — markers: {mix}, recurrence {N}
  2. {topic}  — markers: {mix}, recurrence {N}
  3. {topic}  — markers: {mix}, recurrence {N}
Report: workspace/tmp/kb-gaps-{today}.md
Applied: {N}  (or "skipped — report only")
```

If `--verbose`, also dump the full report inline below the summary.

## Notes

- **Gap ≠ stale.** This skill finds *missing* topics. For stale, duplicate, or orphaned entries use `/nase:kb-review`.
- **Recurrence threshold matters.** Default `--min-recurrence 2` filters one-off events. Drop to `1` for first-pass discovery; raise to `3+` for high-precision flagging.
- **No automatic writes** unless `--auto` is passed. Default flow is propose-then-confirm.
- **False-positive watch:** `lookup`-only clusters and `first_time`-only clusters are noisy by nature — sanity-check before applying. The current marker set deliberately excludes the broad Chinese "原来" because of its "originally" sense.
- **Integration:** `/nase:wrap-up` may invoke `/nase:kb-gap-detect --days 7` weekly as a hygiene complement to `/nase:kb-review`.
