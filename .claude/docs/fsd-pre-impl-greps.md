# Pre-Implementation Greps

Cheap greps that prevent 2-iteration fix cycles in Phase 4. Skip when not applicable to the task type.

- **Adding a parameter to a widely-used constructor or modifying a shared service signature:** grep the test directory for direct instantiation sites (e.g. `new {ServiceName}(`) — not just callers that go through a base/helper class. Direct sites bypass standard wiring and are the most common source of `CS7036` / "missing required parameter" failures. Fix all of them before running tests, not after.
- **Adding a `using` directive for a type that could live in multiple assemblies** (especially generic-sounding names like `QuerySourceType`, `Status`, `Result`, `*Options`): grep `namespace.*{TypeName}|class {TypeName}|enum {TypeName}|interface {TypeName}` in the source first to confirm the actual namespace. The wrong namespace yields `CS0246` on every consumer file — one grep prevents N fixes.
- **TFM / framework version bump (e.g. net8.0 → net10.0, netcoreapp3.1 → net6.0):** before touching `.csproj` / pipeline YAML, grep for hardcoded framework monikers across build artifacts: `grep -rE "net{old}\.0|netcoreapp|net6\.0|net7\.0" --include="*.csproj" --include="*.wixproj" --include="*.ps1" --include="Dockerfile*" --include="*.yml" --include="*.yaml"`. WiX project files and PowerShell build scripts hard-code monikers in publish/copy paths; SDK version bumps alone won't catch them. Sanitized pattern: one missed moniker set required multiple CI rounds because `.wixproj` and `.ps1` publish paths were not updated together.
- **Chained NuGet PRs (consumer-side version bump where producer publish is concurrent):** before pushing the consumer-side bump, confirm the producer's package is on the feed — either (1) wait for the producer's `Push on main` pipeline to finish, (2) `dotnet nuget list source` + restore probe locally, or (3) explicitly note the race in the consumer PR body so reviewers know CI failures are publish-lag, not code. Sanitized pattern: a consumer package bump raced the producer publish pipeline; CI failed until publish completed.
